1 Introdução e Pergunta de Pesquisa

O mercado de dados brasileiro cresceu de forma acelerada entre 2021 e 2024. A pesquisa State of Data Brazil captura, ano a ano, o perfil de quem trabalha com dados no país — suas ferramentas, salários, cargos e distribuição regional. Mas será que o interesse público por essas tecnologias, medido pelo Google, acompanha o que os profissionais realmente usam?

Este trabalho responde às seguintes perguntas:

  1. Linguagens e ferramentas: As tecnologias mais usadas pelos profissionais (State of Data) são também as mais buscadas no Google por estado e ao longo do tempo?
  2. Tendência temporal: O crescimento do uso de Python e SQL entre profissionais (2021–2024) é paralelo ao aumento de buscas no Google?
  3. Distribuição regional: Há estados onde o interesse online supera a presença de profissionais — potenciais mercados emergentes?
  4. Recuperação de perfis (TF-IDF): Dado um perfil de competências desejado, quais profissionais do State of Data são os mais relevantes?

3 Limpeza, Padronização e Integração

O script src/limpeza/integra.py executou as seguintes transformações:

State of Data

  • Extração da sigla UF a partir do formato "São Paulo (SP)" usando expressão regular
  • Conversão das faixas salariais textuais para o ponto médio numérico (ex: "R$4.001 a R$6.000" → 5000.5)
  • Agrupamento de cargos em categorias padronizadas (Analista de Dados, Cientista de Dados, Engenheiro de Dados, etc.)

Google Trends

  • Normalização dos nomes de estado (trata variantes como "State of São Paulo")
  • Remoção de estados sem dados suficientes (interesse = 0)

Integração

Agregação do State of Data por UF × ano e união (inner join) com o interesse médio do Google Trends por grupo de termos, resultando em 95 pares UF×ano com informações de ambas as fontes.

4 Modelo de Recuperação da Informação — TF-IDF

4.1 Justificativa da escolha

Utilizamos o modelo Vetorial com TF-IDF (Term Frequency–Inverse Document Frequency) com similaridade do cosseno. A escolha se justifica por:

  • Adequação ao problema: os perfis dos respondentes são descrições textuais de atributos (linguagem, cargo, estado, experiência), e o TF-IDF captura quais termos distinguem um perfil dos demais
  • Implementação eficiente via scikit-learn, amplamente usada em produção
  • Interpretabilidade dos resultados: os pesos TF-IDF são diretamente legíveis

4.2 Construção do índice

Cada respondente do State of Data é representado como um documento de texto que concatena seus atributos: linguagem principal, cargo, UF, nível de ensino e tempo de experiência.

Código 
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

def perfil_para_texto(row: pd.Series) -> str:
    """Converte atributos de um respondente em texto para o TF-IDF."""
    campos = [
        str(row.get("linguagem_mais_usada", "") or ""),
        str(row.get("cargo_grupo", "") or ""),
        str(row.get("uf", "") or ""),
        str(row.get("nivel_ensino", "") or ""),
        str(row.get("tempo_experiencia_dados", "") or ""),
    ]
    return " ".join(c for c in campos if c and c != "nan").lower()

# Cria o corpus
corpus = df_sod.apply(perfil_para_texto, axis=1)
corpus_valido = corpus[corpus.str.strip() != ""]
idx_validos = corpus_valido.index

# Treina o TF-IDF
vectorizer = TfidfVectorizer(
    ngram_range=(1, 2),   # unigramas e bigramas
    min_df=3,             # ignora termos raros
    max_df=0.95,          # ignora termos ubíquos
)
tfidf_matrix = vectorizer.fit_transform(corpus_valido)

print(f"Corpus: {tfidf_matrix.shape[0]:,} documentos × {tfidf_matrix.shape[1]:,} termos")
print(f"\nExemplos de documentos:")
for doc in corpus_valido.iloc[:3]:
    print(f"  '{doc}'")

4.3 Sistema de busca de perfis

Código 
def buscar_perfis(query: str, n: int = 5) -> pd.DataFrame:
    """
    Dado uma query em texto, retorna os n perfis mais similares
    no State of Data, ordenados por similaridade do cosseno.
    """
    q_vec = vectorizer.transform([query.lower()])
    scores = cosine_similarity(q_vec, tfidf_matrix).flatten()

    top_idx = scores.argsort()[::-1][:n]
    idx_originais = idx_validos[top_idx]

    resultado = df_sod.loc[idx_originais, [
        "ano", "uf", "cargo_grupo", "linguagem_mais_usada",
        "nivel_ensino", "tempo_experiencia_dados", "faixa_salarial"
    ]].copy()
    resultado.insert(0, "similaridade", scores[top_idx].round(3))
    return resultado.reset_index(drop=True)

4.4 Perfis de estados via TF-IDF

Agrupamos os respondentes por estado e calculamos o termo com maior peso TF-IDF por UF — revelando a “identidade tecnológica” de cada estado.

Código 
feature_names = vectorizer.get_feature_names_out()

termos_por_uf = {}
for uf in df_sod["uf"].dropna().unique():
    mask = (df_sod["uf"] == uf) & df_sod.index.isin(idx_validos)
    idx_uf = df_sod.index[mask].intersection(idx_validos)
    if len(idx_uf) == 0:
        continue
    pos_no_corpus = [list(idx_validos).index(i) for i in idx_uf if i in idx_validos]
    if not pos_no_corpus:
        continue
    sub = tfidf_matrix[pos_no_corpus]
    media = np.asarray(sub.mean(axis=0)).flatten()
    top3_idx = media.argsort()[::-1][:3]
    termos_por_uf[uf] = ", ".join(feature_names[top3_idx])

df_termos_uf = (pd.DataFrame.from_dict(termos_por_uf, orient="index",
                                        columns=["termos_tfidf_top3"])
                .reset_index().rename(columns={"index": "uf"})
                .sort_values("uf"))

6 Discussão Crítica

A análise revelou pontos de convergência e divergência entre o que os profissionais de dados usam e o que a população brasileira busca no Google.

Python domina, mas SQL resiste. Python manteve crescimento consistente no State of Data entre 2021 e 2024, acompanhado por interesse crescente no Google Trends, especialmente a partir de 2022 — período que coincide com a explosão de interesse em IA generativa no Brasil. SQL, por outro lado, lidera as buscas no Google apesar de ocupar o segundo lugar entre as linguagens mais usadas, sugerindo que há uma demanda de aprendizado superior à adoção profissional declarada.

Concentração geográfica. Os estados SP, MG e RJ concentram a maior parte dos respondentes, mas estados como CE, RS e SC apresentam interesse no Google Trends próximo ou igual ao de estados maiores, sugerindo mercados em crescimento com potencial de absorção de profissionais.

Limitações. O Google Trends fornece índices relativos (0–100), não volumes absolutos, o que dificulta comparações diretas entre estados de populações muito diferentes. A tabela de instrução do IBGE não foi incluída por indisponibilidade da API no período de 2021–2022. O modelo TF-IDF não captura sinônimos nem contexto semântico — um modelo de embeddings densos poderia melhorar a qualidade das buscas de perfis.

7 Conclusão

Este trabalho integrou dados de 17.426 respondentes do State of Data Brazil (2021–2024) com 4.770 registros do Google Trends para 15 termos de tecnologia, cobertos por estado brasileiro e período anual.

Os principais achados são:

  • Python cresceu de ~45% para ~55% de adoção entre profissionais, com curva de interesse no Google Trends acompanhando o movimento, especialmente após 2022.
  • SQL lidera as buscas online mas ocupa o segundo lugar na adoção profissional, revelando uma demanda de aprendizado maior do que a declaração de uso principal.
  • O modelo TF-IDF permitiu construir um sistema de recuperação de perfis profissionais que responde a consultas em linguagem natural, identificando os respondentes mais alinhados a um conjunto de competências.
  • A correlação entre adoção de Python por estado (State of Data) e interesse em linguagens no Google Trends é positiva, porém moderada, refletindo diferenças de perfil demográfico e acesso tecnológico entre regiões.
